## 백준 12015 가장 긴 증가하는 부분 수열 2

### 문제 설명

https://www.acmicpc.net/problem/12015

수열 A가 주어졌을 때, 가장 긴 증가하는 부분 수열을 구하는 프로그램을 작성하시오.

예를 들어, 수열 A = {10, 20, 10, 30, 20, 50} 인 경우에 가장 긴 증가하는 부분 수열은 A = {10, 20, 10, 30, 20, 50} 이고, 길이는 4이다.

#### 입력
첫째 줄에 수열 A의 크기 N (1 ≤ N ≤ 1,000,000)이 주어진다.

둘째 줄에는 수열 A를 이루고 있는 Ai가 주어진다. (1 ≤ Ai ≤ 1,000,000)

#### 출력
첫째 줄에 수열 A의 가장 긴 증가하는 부분 수열의 길이를 출력한다.

### 문제 접근

단계별 풀어보기를 통해서 문제를 접했기 때문에 이분탐색을 활용해야한다는 사실을 알고 시작했다. 하지만... 어디에 이분탐색을 써야하는지 잘 감이 안 와서 질문하기 게시판의 내용을 조금 참고해서 "가장 긴 증가하는 부분 수열 길이를 구하기 위한 배열을 관리"해야한다는 사실을 알게 되었다.

풀이 방식을 간단히 정리하면

1. 가장 긴 증가하는 부분 수열 길이를 구하기 위한 배열을 관리한다. (이 배열을 항상 오름차순으로 정렬되어 있음)
2. 순차적으로 주어진 수열의 값을 보면서 해당 배열을 업데이트한다.
    ```
    if 새로 들어온 수가 마지막 수보다 큰 경우
        수열의 마지막에 추가한다 (오름차순 정렬이 유지)
    else 새로 들어온 수가 마지막 수보다 작은 경우
        수열 내부에서 들어오고자 하는 수보다 같거나 큰 수 중 가장 작은 수를 해당 수로 대치한다. (여기서도 오름차순으로 정렬이 흐트러지지 않음)
    ```

대치하는 과정을 통해서 뒤쪽에 등장하는 새로운 수들을 더 받을 수 있도록 해서 가장 "긴" 증가 수열의 길이를 구할 수 있다.

하지만 주의할 점은 내가 관리하고 있는 배열의 수들이 실제 가장 긴 증가하는 부분 수열의 값들을 아닐 수 있다는 점이다!! (이렇게 계산하는 알고리즘이 길이를 구하는 것에는 영향을 미치지 않지만, 실제 해당 배열이 우리가 찾고자하는 부분 수열 자체는 아닌 경우가 존재한다!)
